创建自己私有pod

发表于 2019-01-21 | 分类于 iOS

1、注册CocoaPods

不确定自己是否注册的可以使用命令查看

  • pod trunk me

如果没有注册，输入一下命令

  • pod trunk register 你的邮箱 ‘你的名字’ –verbose（加上--verbose可以查看详细信息,方便出错时查看）

注册完成之后会给你的邮箱发个邮件,进入邮箱邮件里面有个链接,需要点击确认一下。

注册完成后使用pod trunk me检验注册是否成功

4、本地代码创建成功，修改podspecs索引文件

cd到本地代码文件夹

修改.podspec文件

  • s.name：名称，pod search搜索的关键词,注意这里一定要和.podspec的名称一样,否则报错
  • s.version：版本号
  • s.author:作者
  • s.homepage:项目主页地址，文章开头创建的github地址
  • s.summary: 项目简介
  • s.source:项目源码所在地址，文章开头创建的github地址
  • s.license:许可证 s.license = { :type => 'MIT', :file => 'LICENSE' }
  • s.platform:项目支持平台
  • s.requires_arc: 是否支持ARC
  • s.source_files:需要包含的源文件s.source_files = 'PrivatePodDemo/Classes/**/*'
  • s.public_header_files:需要包含的头文件
  • s.ios.deployment_target:支持的pod最低版本
  • s.dependency:依赖库，不能依赖未发布的库

5、验证自己的podspec

提交代码到远程仓库

依次输入命令

  • git add .
    *git commit -m “commit message”
    git remote add origin 远程仓库代码地址
    git push origin master
    git tag 版本号
    git push –tag

使用一下命令，确认自己的podspec文件语法不存在错误

  • pod spec lint PrivatePodDemo.podspec --verbose

这个时候你可能会出现这样的错误

莫慌，这是因为远端还没有0.0.1分支，出现原因可能是tag没有成功上传到远端。解决办法：依次输入下面命令

  • git tag -m “pod spec” “0.0.1”
  • git push origin –tags

开始验证

  • pod spec lint PrivatePodDemo.podspec

最终出现MboxAppCommon.podspec passed validation即为验证通过

7、发布

执行命令

  • pod trunk push PrivatePodDemo.podspec

注意这一步比较慢,大概5到10分钟吧。到这里自己的私有pod库就制作完成了。

8、更新和维护

修改完自己的代码以后推送到远端，切记修改podspec文件中的版本号，并重新打tag上传，在进行新一轮的验证和发布。
